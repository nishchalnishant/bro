# Error Notebook — NEET PG Preparation

---

## How to Use This Error Notebook

**Purpose**: Every wrong answer from every mock test, GT, and PYQ session gets logged here. The goal is zero repeat mistakes.

**Rules**:
1. Log every wrong answer within 24 hours of test analysis
2. Do not skip any wrong answer — especially the ones that "almost" make sense
3. Log uncertain correct answers too — mark them with a `?` next to Error Type
4. Review this notebook every Sunday (Weekly Review Protocol in Section 4)
5. If a fact appears in this notebook more than twice, it goes on the "Never Again" list (Section 8)

**What this notebook is not**: A collection of interesting facts. It is a diagnostic tool for mistakes. Every entry must include an action (the Key Learning).

## 3. Error Categorization System

Assign exactly one category to each wrong answer. Be honest — miscategorizing errors prevents fixing the right problem.

| Error Type | Definition | What It Means | What to Do |
|-----------|-----------|--------------|-----------|
| **Concept Gap** | You did not know or misunderstood the underlying concept | Foundation is missing for this topic | Return to the source material (textbook/video). Re-study the concept, not just the fact |
| **Recall Gap** | You knew the concept but could not retrieve the specific fact, value, classification, or name | Encoding is weak — needs reinforcement | Add to flashcard. Revise at 1 day, 3 days, 1 week, 1 month |
| **Silly Mistake** | You knew the answer but misread the question, misread an option, calculated incorrectly, or clicked the wrong choice | Attention/process error, not knowledge error | Note the pattern. Create a pre-test checklist (e.g., "always underline EXCEPT") |
| **Trap** | The question was designed to mislead — a distractor that tests deep understanding or pattern recognition | Expected — these differentiate top rankers | Note exactly what the trap was. Do not panic. These become your edge over time |
| **Never Studied** | The topic was completely outside your preparation — not just forgotten, but never encountered | Coverage gap | Evaluate: is this high-yield? If yes, add chapter to study plan. If no, note and skip |

### Common Mistake: Misclassifying Recall Gap as Concept Gap

If you could have answered correctly after seeing the answer and immediately understood it — that is a **Recall Gap**, not a Concept Gap. Concept Gaps require re-study. Recall Gaps require repetition.

## 4. Weekly Error Review Protocol

**When**: Every Sunday, 30–45 minutes

**Process**:
1. Open the error notebook to entries from the past week
2. Cover the "Key Learning" column and try to recall the answer for each entry
3. Mark each entry: Retained (Y) or Not Retained (N)
4. All entries marked N go into the next week's active review pile
5. Update the Error Frequency Tracker (Section 5) with new entries
6. Move any fact that appears 3+ times to the "Never Again" list (Section 8)

**Rule**: Do not add new entries during the weekly review. The review session is for retrieval only, not input.

## 7. Monthly Error Audit

**When**: Last day of each month, 60–90 minutes

**Instructions**:
1. Open the Error Frequency Tracker and count total entries per subject this month
2. Identify the top 3 subjects with the most errors — these define next month's revision priority
3. Calculate retention rate per subject: (Entries marked Y / Total entries reviewed) × 100
4. Any subject below 60% retention rate needs a method change — not more time, a different method
5. Review all "Never Again" list entries (Section 8). Every entry must be tested verbally
6. Archive this month's analysis below with a date stamp
