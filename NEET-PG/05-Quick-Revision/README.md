# 05 — Quick Revision

This folder contains condensed, high-yield revision material designed for rapid review in the days and hours before the exam. It is not a learning resource — it assumes the subject has already been studied. Use it to reinforce memory, recall mnemonics, and run through one-liners and comparison tables at speed. Everything here is optimized for retention, not depth.

## Files

| File | Description |
|------|-------------|
| `Last-Day-Notes/last-day-notes.md` | Ultra-condensed subject-wise notes covering the most testable facts — intended for the final 24 hours before the exam |
| `Mnemonics/master-mnemonics.md` | Master list of mnemonics organized by subject for fast recall of lists, drug classes, and classification criteria |
| `One-Liners/master-one-liners.md` | Single-sentence high-yield facts across all subjects — ideal for rapid-fire revision in short time blocks |
| `High-Yield-Tables/comparison-tables.md` | Side-by-side comparison tables for commonly confused conditions, drugs, and investigations |
| `Normal-Values/normal-values.md` | Comprehensive normal lab values and clinical cutoffs organized by system — hematology, biochemistry, ABG, renal, cardiac, CSF, obstetrics/pediatrics, and key diagnostic thresholds with PYQ hotspot markers |

## Revision Strategy

- **Daily use:** Start every study session with 10–15 minutes on `One-Liners/master-one-liners.md` to keep high-yield facts active in working memory throughout the preparation period.
- **Weekly use:** Review `High-Yield-Tables/comparison-tables.md` once per week to sharpen differential diagnosis thinking and prevent confusion between similar conditions.
- **Final week:** Switch to a mnemonics-first review — spend each morning running through `Mnemonics/master-mnemonics.md` for the subjects covered that day before opening any notes.
- **Day before exam:** `Last-Day-Notes/last-day-notes.md` is the only file to read the evening before the exam — avoid new material; use it purely to activate what is already stored.
- **Starting point for daily use:** The recommended daily driver lives one folder up — see [`../00-Master-Index/DAILY-REVISION.md`](../00-Master-Index/DAILY-REVISION.md) for the structured daily revision protocol that integrates all quick-revision resources.
